### 一、 什么是 MCP？（三分钟大白话讲透原理）

#### 1. 核心思想：大模型的“USB-C 物理接口”
在没有 MCP 之前，如果想让大模型帮你干活，你必须针对每个模型（GPT-4、Claude、DeepSeek）写一套定制的 API 接口代码。
而 **MCP（Model Context Protocol）** 是 Anthropic 发布的开源标准。它就像是 AI 领域的 **“USB-C 统一接口”**：
* 你的 Rust 软件只需要实现一个标准的 **MCP Server**；
* 任何支持 MCP 的 AI 客户端（如 Claude Desktop、Cline 插件等）就能像插入 U 盘一样，**瞬间无缝读懂并直接控制你的几何排版画布**！

#### 2. MCP 的极简三步握手协议
MCP 在底层的通信采用的是极其轻量、标准的 **JSON-RPC 2.0 协议**。大模型和你的 Rust 服务之间，只玩一个“三步游戏”：
1. **Handshake（握手）**：客户端连上你的服务，互相问好并协商协议版本。
2. **List Tools（亮出家伙）**：AI 客户端问：*“你有哪些本领（API 函数）？”* 你的 Rust 服务返回一个 JSON 数组，里面用 **JSON Schema** 格式写着所有你支持的排版工具（比如 `solve_layout`）和每个工具需要的参数描述。
3. **Call Tool（调用执行）**：当用户在 AI 对话框打字说“把 3 号元素居中”。AI 自动匹配你的工具列表，发来一个请求：*“请帮我执行 `align_element(id: 3, type: "Center")`”*。你的 Rust 运行几何计算，并把结果吐回给 AI。

#### 3. ⚠️ 铁律中的铁律：绝不能使用 `println!` 
在本地测试中，大模型和你的 Rust MCP 服务主要通过 **标准输入输出（Stdio）** 进行数据交互。
* **这意味着：你的 `stdout` 就是通信的黄金物理通道。**
* **铁律**：**在你的 MCP 代码里，绝对、严禁、100% 不能使用 `println!` 来打印任何调试信息！** 如果你用了 `println!`，调试字符会直接混进 JSON-RPC 的数据流中，直接导致大模型断开连接、程序崩溃。
* **解决**：所有的调试日志、打印，必须使用 `eprintln!`（输出到 `stderr`，大模型会默默过滤掉它），或者使用 Rust 官方的 `tracing` 日志框架并定向到 `stderr`。

use schemars::JsonSchema;
use serde::{Deserialize, Serialize};

use kurbo::BezPath;
use shape_layout::{layout_rows, LayoutElement, LayoutConfig, LayoutSolution, SizeStrategy};

// ═══════════════════════════════════════════════════════════════
// 数据交互模型 (自动导出为 JSON Schema 供 AI 识别)
// ═══════════════════════════════════════════════════════════════

/// AI 在改变元素排版时发送的输入参数
#[derive(Debug, Deserialize, JsonSchema)]
pub struct UpdateElementParams {
    /// 目标元素的 ID
    pub id: String,
    /// 偏好宽度 (mm)
    pub width: f64,
    /// 偏好高度 (mm)
    pub height: f64,
    /// 尺寸策略："Fixed"（固定宽，不可/可缩）或 "Fill"（弹性拉伸）
    #[schemars(description = "Sizing strategy: 'Fixed' or 'Fill'")]

// MCP 服务器状态与逻辑实现
// ═══════════════════════════════════════════════════════════════

/// MCP 服务器的状态
struct McpServerState {
    /// 模拟内存中的画布状态
    pub elements: Mutex<Vec<LayoutElement>>,
    pub config: Mutex<LayoutConfig>,
    pub container: Mutex<BezPath>,
}

#[derive(Clone)]
struct McpServer {
    state: std::sync::Arc<McpServerState>,
}

// 利用 rmcp 宏，一键生成工具路由器
#[tool_router(server_handler)]
impl McpServer {
    
    /// 工具 1：供 AI 读取当前的画布排版状态（包括元素、全局间距对齐等）
    #[tool(description = "读取当前画布上的所有元素、全局边距和对齐方式")]
    async fn read_canvas_state(&self) -> Result<String, String> {
        let elements = self.state.elements.lock().unwrap();
        let config = self.state.config.lock().unwrap();
        
        let state_json = serde_json::json!({
            "elements": *elements,
            "config": *config,
        });
        
        // 记住：只能用 eprint! 做调试打印，不要用 print!
        eprintln!("AI 抓取了当前画布状态");
        Ok(state_json.to_string())
    }

    /// 工具 2：供 AI 动态添加、更新某个元素的尺寸和拉伸策略
    #[tool(description = "添加或更新画布上某个元素的宽度、高度和拉伸策略")]
    async fn update_element(&self, params: UpdateElementParams) -> Result<String, String> {
        let mut elements = self.state.elements.lock().unwrap();
        
        let strategy = match params.size_strategy.as_str() {
            "Fill" => SizeStrategy::Fill,
            "FixedShrink" => SizeStrategy::Fixed { shrinkable: true },
            _ => SizeStrategy::Fixed { shrinkable: false },
        };

        // 查找并更新，如果不存在则新建
        if let Some(elem) = elements.iter_mut().find(|e| e.id == params.id) {
            elem.width = params.width;
            elem.height = params.height;
            elem.constraints.size_strategy = strategy;
            eprintln!("AI 更新了元素: {}", params.id);
        } else {
            let mut new_elem = LayoutElement::new(&params.id, params.width, params.height);
            new_elem.constraints.size_strategy = strategy;
            elements.push(new_elem);
            eprintln!("AI 新增了元素: {}", params.id);
        }

        Ok(format!("元素 {} 状态更新成功", params.id))
    }

    /// 工具 3：供 AI 命令你的 Rust 引擎立刻执行数学求解
    #[tool(description = "调用 shape_layout 核心引擎进行高精度的几何拼版求解，并返回每个元素的绝对 X/Y 坐标")]
    async fn compile_layout(&self) -> Result<String, String> {
        let elements = self.state.elements.lock().unwrap();
        let config = self.state.config.lock().unwrap();
        let container = self.state.container.lock().unwrap();

        // 直接调用你的 1D 扫描线 + kasuari 约束大脑！
        let solution = layout_rows(&container, &elements, &config);
        
        eprintln!("几何排版编译器求解完成。已排入：{}", solution.placed_count());
        Ok(serde_json::to_string(&solution).unwrap())
    }
}

    tracing_subscriber::fmt()
        .with_writer(std::io::stderr)
        .init();

    let initial_heart = heart_shape(0.0, 0.0, 20.0); // 假装已有心形生成函数

    server_handle.waiting().await?;
    Ok(())
}

### 1. MCP 到底是什么？（大白话版）

MCP（Model Context Protocol）是 **AI 助手和外部工具之间沟通的标准协议**（类似 OpenAI Function Calling 的升级版 + 标准化）。

它解决的核心问题是：
- AI 想调用你的 `shape_layout` 排版功能，但需要一种**结构化、安全、可发现**的方式。
- 不再是乱七八糟的 Prompt Engineering，而是 AI 可以“看到工具列表”、调用工具、拿到结果、继续思考的**标准化流程**。

**典型流程**：

### 🧩 核心拼图：MCP 到底是什么？

MCP（模型上下文协议）本质上是一套**标准化的通信规则**，它让AI（客户端）能以统一的方式去调用外部工具（服务器）。

在你的场景里，运作流程非常清晰：

*   **MCP 服务器 (Server)**：一个独立的服务程序，它**注册并暴露**`shape_layout`的核心排版能力，作为一个或多个**工具 (Tool)**，供AI调用。

### 🔗 完整的工作流程

把这一切串起来，一个完整的工作流就清晰了：

1.  **你的MCP服务器启动**，通过`stdio`或`HTTP`等待连接。
2.  **AI客户端（如支持MCP的工具）连接并初始化**，发送`InitializeRequest`建立通信。
3.  **客户端发送`ListToolsRequest`**，你的服务器返回注册的`layout`工具及其描述和参数JSON Schema。
4.  **用户下达任务**，你的客户端程序将任务（包含`layout`工具说明）发给DeepSeek V4 Pro。
5.  **DeepSeek返回**一个`CallToolRequest`，要求调用`layout`工具，参数为`{container: "...", elements: [...]}`。
6.  **你的客户端将此请求转发给你的MCP服务器**。
7.  **你的MCP服务器解析请求**，调用`shape_layout`的`layout_rows`函数。
8.  **服务器将计算出的`LayoutSolution`**（精确坐标）作为成功结果返回给客户端。
9.  **客户端将结果提交给DeepSeek**，由它组织最终的自然语言回复，或直接用于后续操作。

## 🧭 MCP 是什么？用最简单的话说

```
传统 AI 调用工具：
┌─────────┐    "帮我算一下 2+3"    ┌─────────┐

```

四、DeepSeek Function Calling 怎么跟 MCP 对接？

DeepSeek 的 Function Calling 格式：

{
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "create_container",
        "description": "创建一个异形容器",
        "parameters": {
          "type": "object",
          "properties": {
            "shape_type": {
              "type": "string",
              "enum": ["heart", "circle", "rect", "custom"]
            },
            "width": {"type": "number"},
            "height": {"type": "number"}
          },
          "required": ["shape_type", "width", "height"]
        }
      }
    }
  ]
}


MCP Server 这边（rmcp）：

#[tool]
async fn create_container(
    shape_type: String,
    width: f64,
    height: f64,

    // 调用 shape_layout 创建容器
    let container = Container::new(shape_type, width, height);
    Ok(container.id.to_string())
}
