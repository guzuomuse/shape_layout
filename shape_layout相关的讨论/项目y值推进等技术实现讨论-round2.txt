## 🔬 "所有元素挤在右侧"的真正根因

### 根因 1：`find_bottleneck` 对对称形状的缺陷

```
心形容器，Y = -150（分叉处）：
  左心房：(-308, -9.6)，宽度 298.6
  右心房：(9.6, 308.2)，宽度 298.6

find_bottleneck 处理左心房：
  轮廓点：(-308, y1), (-200, y2), ..., (-9.6, yn)
  边缘方向分析 → safe_left = -308, safe_right = -9.6
  → 区间 (-308, -9.6)

find_bottleneck 处理右心房：
  轮廓点：(9.6, y1), (100, y2), ..., (308, yn)
  边缘方向分析 → safe_left = 9.6, safe_right = 308
  → 区间 (9.6, 308)

问题：左心房的区间 (-308, -9.6) 和右心房的 (9.6, 308) 之间
      有一个"鸿沟"：(-9.6, 9.6)
      
      但心形的实际形状在 Y=-150 处是连续的！
      中间应该有一个连接部分（心形底部尖角向上延伸的部分）
      
      为什么 find_bottleneck 返回了两个分离的区间？
```

**答案：`i_overlay` 的 `Intersect` 运算返回了两个独立的 contour（左心房和右心房），`find_bottleneck` 对每个 contour 独立计算瓶颈，所以返回了两个分离的区间。**

这是**正确的行为**，因为心形在 Y=-150 处确实有两个不连通的"安全区域"（左右心房）。

### 根因 2：`max_by` 的确定性 + 对称性

```rust
let widest = row_range.intervals.iter().max_by(|a, b| {
    (a.1 - a.0).partial_cmp(&(b.1 - b.0)).unwrap_or(Equal)
});
```

当左右心房宽度**完全相等**时（心形是对称的），`max_by` 返回**第一个**元素。

`row_range.intervals` 的排序来自 `get_intervals_at` 最后的 `sort_by(|a, b| a.0.partial_cmp(&b.0))`——按左边界升序排列。

所以：
- 左心房 (-308, -9.6) 的左边界 = -308
