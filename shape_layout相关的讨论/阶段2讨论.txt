一、关于那两个关键问题的裁决

✅ 问题 1：Stretch 时 HAlign 的行为

“当有 stretchable 元素时，行会被撑满整个区间，HAlign 实际上被覆盖，这是预期的吗？”

大哥的裁决：✅ 这是完全正确的行为，不需要新增 HAlign::Stretch。

理由有三：

1. 语义清晰  
   • HAlign::Left/Center/Right = “内容在可用空间内怎么摆”  

   • stretchable = true = “我愿意吃掉剩余空间”

2. 工业排版惯例  
   • InDesign / Figma / CSS 都是这个逻辑：  

     ◦ justify-content: space-between ≠ flex-grow: 1  

     ◦ 你现在是 grow 行为，不是 justify 行为

3. 避免状态爆炸  
   • 如果再加 HAlign::Stretch，会和 Left/Center/Right 互斥，反而增加心智负担

✅ 正确做法：
• 只要行内有 任一元素 stretchable → solver 强制 right_last - left_0 == interval_width

• 此时 HAlign 逻辑上失效，但你不需要报错，也不用处理

✅ 问题 2：Baseline 的默认值

“元素不指定 baseline 时，默认用 height（底部），这样可以吗？”

大哥的裁决：✅ 可以，而且这是最稳妥的默认。

原因：

baseline 值 视觉结果 是否符合直觉

None → height 底部对齐 ✅ 符合“没说基线就按底部算”

Some(0.0) 顶部对齐 ✅ 用于图标 / 图形

Some(height * 0.8) 文字基线 ✅ 用于文本

这样你未来可以支持：
LayoutElement::text("Hello", 20.0).with_baseline(16.0); // 文字
LayoutElement::icon(24.0).with_baseline(0.0);          // 图标


✅ 向后兼容，语义干净。

一、关于那两个关键问题的裁决

❓ 问题 1：Stretch 时 HAlign 怎么办？

“当有 stretchable 元素时，行会被撑满整个区间，此时 HAlign 实际上被覆盖了。这是预期的吗？”

✅ 大哥的裁决：这是预期行为，但语义要改。

原因：
• 在 Figma / CSS 里，justify-content: space-between 和 align-items: stretch 是两个维度的概念。

• 你的 HAlign 是主轴对齐（X 轴）。

• Stretch 是交叉轴 / 尺寸策略，不是对齐方式。

✅ 正确做法（请立刻采纳）

不要复用 HAlign。

新增一个尺寸策略枚举：
pub enum SizeStrategy {
    /// 用给定尺寸（默认）
    Fixed,
    /// 包裹内容（未来）
    Hug,
    /// 撑满可用宽度
    Fill,
}


然后：
pub struct LayoutElement {
    pub size_strategy: SizeStrategy,
    pub width: f64,
    pub min_width: Option<f64>,
    pub max_width: Option<f64>,
}


kasuari 逻辑变为：
for elem in row_elements {
    match elem.size_strategy {
        SizeStrategy::Fixed => {
            // REQUIRED 宽度
        }
        SizeStrategy::Fill => {
            // WEAK 宽度 + STRONG 撑满区间
        }
    }
}


✅ 这样语义干净，不再打架。

❓ 问题 2：Baseline 的默认值

“元素不指定 baseline 时，默认用 height（底部）作为 baseline，这样可以吗？”

✅ 大哥的裁决：可以，但必须区分“文字”和“非文字”。

原因：
• 文字的 baseline ≠ 矩形的底部

• 图标 / 路径 / 图片 没有 baseline，只有 bottom

✅ 正确做法

pub struct LayoutElement {
    /// 从元素顶部到基线的距离
    /// None = 使用元素高度（矩形底部）
    pub baseline: Option<f64>,
}


计算规则（在 layout_rows 放置时）：
let baseline_offset = match elem.baseline {
    Some(b) => b,
    None => elem.height, // 默认：矩形底部
};

let y = row_y + row_baseline - baseline_offset;


✅ 这样：
• 文字元素：指定 baseline = ascent

• 图标 / 矩形：不指定 → 底部对齐

• 混合排版时，基线对齐正确
