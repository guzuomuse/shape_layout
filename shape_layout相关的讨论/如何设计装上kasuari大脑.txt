### 三、 `shape_layout` 内部的“大脑装配与求解流水线”

当 `solve(doc: LayoutDocument)` 被调用时，`shape_layout` 内部按以下 **5 个物理阶段** 顺畅执行：

```
1. 解析 LayoutDocument
       │
       ▼
2. 几何切片阶段 (RangeGenerator)
   - 自上而下，通过 max(height) 计算每一行的 H_row 和 Y_current
   - 拿 (Y_current, H_row) 调用你的扫描线算法，算得该行的物理安全边界 [L, R]
       │
       ▼
3. kasuari 变量初始化 (Variable Registration)
   - 对每个 Element，在 kasuari 求解器里注册 4 个变量: left, right, top, bottom
       │
       ▼
4. 约束注入阶段 (Constraint Injection)
   - 注入物理边界：solver.add_constraint(left >= L) / (right <= R)
   - 注入行内排流：solver.add_constraint(A.right + gap == B.left)
   - 注入美学约束：自动翻译 LayoutConstraint 对应的代数方程并注入
       │
       ▼
5. 矩阵求解与收割 (Solve & Harvest)
   - 求解器在微秒内解出所有变量的浮点数值
   - 组装成 ResolvedEntity 数组返回
```

### 四、 `shape_layout` 核心求解器的 Rust 代码草图

为了让你看清 `kasuari` 是如何轻量、优美地作为大脑接入的，下面是核心 Solver 的实现逻辑：

```rust
use kasuari::{Solver, Variable, Expression, Strength};

pub struct ResolvedEntity {
    pub id: u32,
    pub x: f64,
    pub y: f64,
    pub width: f64,
    pub height: f64,
}

pub fn solve_layout(doc: LayoutDocument) -> Result<Vec<ResolvedEntity>, String> {
    let mut solver = Solver::new();
    let mut vars_map = std::collections::HashMap::new();

    // 1. 变量注册与基本约束
    // 对每一个元素，我们为它在 kasuari 中注册几何边界变量
    for row in &doc.rows {
        for elem in &row.elements {
            let left = Variable::new();
            let right = Variable::new();
            let top = Variable::new();
            let bottom = Variable::new();

            // 注册基本恒等式约束：left + width == right
            solver.add_constraint(right.eq(left + elem.width_policy_value()))
                .map_err(|e| e.to_string())?;
            
            // 注册基本恒等式约束：top + height == bottom
            solver.add_constraint(bottom.eq(top + elem.height))
                .map_err(|e| e.to_string())?;

            vars_map.insert(elem.id, (left, right, top, bottom));
        }
    }

    // 2. 运行你的 RangeGenerator，自上而下注入物理切片约束 [L, R]
    let range_gen = RangeGenerator::new(doc.outer_shell)?;
    let mut y_current = doc.padding_top;

    for row in &doc.rows {
        // 计算行高 (最高元素决定)
        let h_row = row.elements.iter()
            .map(|e| e.height)
            .fold(0.0, f64::max);

        // 调用你的扫描线算法拿到安全的 L 和 R
        let row_range = range_gen.get_intervals_at(y_current, h_row, None);
        
        // 假设我们只取第一个最宽的可用安全区间 (复杂的多区间避让可在后期扩展)
        if let Some(&(l, r)) = row_range.intervals.first() {
            // 注入物理边界：限制这一行里的所有元素，绝不能超出物理切片的左右边界 [L, R]
            for elem in &row.elements {
                let &(left, right, top, bottom) = vars_map.get(&elem.id).unwrap();
                
                // left >= L (物理边界左墙)
                solver.add_constraint(left.ge(l)).unwrap();
                // right <= R (物理边界右墙)
                solver.add_constraint(right.le(r)).unwrap();
                
                // 限制 Y 轴位置
                solver.add_constraint(top.eq(y_current)).unwrap();
            }
        }

        // 注入行内流式排布约束 (A.right + gap == B.left)
        for window in row.elements.windows(2) {
            let elem_a = &window[0];
            let elem_b = &window[1];
            let &(_, right_a, _, _) = vars_map.get(&elem_a.id).unwrap();
            let &(left_b, _, _, _) = vars_map.get(&elem_b.id).unwrap();

            solver.add_constraint(left_b.eq(right_a + row.gap_between_elements)).unwrap();
        }

        // 递进 Y
        y_current += h_row + 2.0; // 2.0 为行间距
    }

    // 3. 注入用户声明的美学全球相对约束
    for constraint in &doc.global_constraints {
        match constraint {
            LayoutConstraint::AlignCenterX { id_a, id_b } => {
                let &(left_a, right_a, _, _) = vars_map.get(id_a).unwrap();
                let &(left_b, right_b, _, _) = vars_map.get(id_b).unwrap();
                // 约束方程：(left_a + right_a)/2 == (left_b + right_b)/2
                solver.add_constraint((left_a + right_a).eq(left_b + right_b)).unwrap();
            }
            // ... 翻译其他 LayoutConstraint 规则并注入 ...
        }
    }

    // 4. 收割计算结果
    let mut resolved = Vec::new();
    for (id, (left, right, top, bottom)) in vars_map {
        resolved.push(ResolvedEntity {
            id,
            x: solver.get_value(left),
            y: solver.get_value(top),
            width: solver.get_value(right) - solver.get_value(left),
            height: solver.get_value(bottom) - solver.get_value(top),
        });
    }

    Ok(resolved)
}
```

### 三、 第三步：实现核心求解器 `ShapeSolver`

这是 `kasuari` 和你的“扫描线算法”真正合体的地方。

```rust
pub struct ShapeSolver {
    solver: kasuari::Solver,
    range_gen: RangeGenerator, // 你移植好的 iooverlay 扫描线引擎
}

impl ShapeSolver {
    pub fn solve(&mut self, items: &mut [LayoutItem]) {
        // 1. 初步逻辑求解：先按用户意图算出 Ideal 坐标
        self.solver.solve().unwrap();

        // 2. 几何校准循环：
        let mut current_y = 0.0;
        for item in items {
            // 获取 kasuari 算出的理想 Y 和高度
            let ideal_y = self.solver.get_value(item.y_var);
            let height = self.solver.get_value(item.height_var);

            // 关键：拿着 Y 去探测扫描线区间 [L, R]
            let row_range = self.range_gen.get_intervals_at(ideal_y, height, None);

            if let Some(interval) = self.find_best_interval(row_range, item) {
                // 3. 动态注入物理约束！
                // 给 solver 增加临时约束：x >= L 且 x + width <= R
                self.solver.add_constraint(
                    item.x_var.get_expression() >= interval.0
                ).with_strength(kasuari::Strength::REQUIRED);
                
                self.solver.add_constraint(
                    (item.x_var.get_expression() + item.width_var.get_expression()) <= interval.1
                ).with_strength(kasuari::Strength::REQUIRED);
            }
        }

        // 4. 终极求解：得出物理+逻辑的双重最优解
        self.solver.solve().unwrap();
    }
}
```

```rust
    pub priority: u32,
    /// 元素类型（影响行高计算）
    pub kind: ElementKind,
    /// 约束规则
    pub constraints: ElementConstraints,
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum ElementKind {
    Text,       // 需要基线对齐
    Image,      // 需要中线或顶线对齐
    Path,       // 任意轮廓

    pub align: AlignConstraint,

    pub stay_near: Option<StayConstraint>,
```

```rust
// src/layout/solver.rs

use kasuari::{Solver, Variable, Constraint, Strength, WeightedRelation};
use crate::types::*;

/// kasuari 包装器：将 VDM 约束翻译为 kasuari 约束
pub struct LayoutSolver {
    solver: Solver,
    variables: Vec<LayoutVariable>,
}

/// 布局变量映射
struct LayoutVariable {
    element_id: String,
    x: Variable,
    y: Variable,
    w: Variable,
    h: Variable,
}

impl LayoutSolver {
    pub fn new() -> Self {
        Self {
            solver: Solver::new(),
            variables: Vec::new(),
        }
    }
    
    /// 为一行内的元素建立求解
    pub fn solve_row(
        &mut self,
        row: &RowRange,
        elements: &[&Element],
        placements: &[Option<Placement>],  // 已放置的元素（避免重叠）
    ) -> Result<Vec<Placement>, SolverError> {
        
        self.solver.reset();
        self.variables.clear();
        
        // 1. 为每个元素创建变量
        for elem in elements {
            let vars = LayoutVariable {
                element_id: elem.id.clone(),
                x: self.solver.add_variable(),
                y: self.solver.add_variable(),
                w: self.solver.add_variable(),
                h: self.solver.add_variable(),
            };
            self.variables.push(vars);
        }
        
        // 2. 添加强约束：元素必须在可用区间内
        for (i, elem) in elements.iter().enumerate() {
            let v = &self.variables[i];
            let size = elem.effective_size();  // 考虑旋转后的尺寸
            
            // 宽度/高度固定或范围
            match &elem.constraints.width {
                SizeConstraint::Fixed(w) => {
                    self.solver.add_constraint(
                        Constraint::new(v.w, WeightedRelation::EQ, *w)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                }
                SizeConstraint::MinMax(min, max) => {
                    self.solver.add_constraint(
                        Constraint::new(v.w, WeightedRelation::GE, *min)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                    self.solver.add_constraint(
                        Constraint::new(v.w, WeightedRelation::LE, *max)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                }
                // Auto：由内容决定，这里先设为最小值，后面用弱约束优化
                _ => {}
            }
            
            // 必须在某个可用区间内
            // kasuari 不能直接表达"OR"（在区间A OR 区间B）
            // 策略：外层策略已选择区间，这里只约束在选定区间内
            let interval = self.select_interval(elem, &row.intervals);
            self.solver.add_constraint(
                Constraint::new(v.x, WeightedRelation::GE, interval.0)
                    .with_strength(Strength::REQUIRED)
            ).unwrap();
            self.solver.add_constraint(
                Constraint::new(v.x + v.w, WeightedRelation::LE, interval.1)
                    .with_strength(Strength::REQUIRED)
            ).unwrap();
        }
        
        // 3. 元素间不重叠（强约束）
        for i in 0..elements.len() {
            for j in (i+1)..elements.len() {
                let vi = &self.variables[i];
                let vj = &self.variables[j];
                
                // xi + wi + gap <= xj  OR  xj + wj + gap <= xi
                // 同样，kasuari 不能直接表达 OR
                // 策略：外层策略已决定顺序，这里只约束方向
                let order = self.determine_order(i, j, elements);
                if order == Order::LeftToRight {
                    self.solver.add_constraint(
                        Constraint::new(vi.x + vi.w, WeightedRelation::LE, vj.x - 2.0)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                } else {
                    self.solver.add_constraint(
                        Constraint::new(vj.x + vj.w, WeightedRelation::LE, vi.x - 2.0)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                }
            }
        }
        
        // 4. 对齐约束（强或弱）
        for (i, elem) in elements.iter().enumerate() {
            let v = &self.variables[i];
            match elem.constraints.align.vertical {
                VAlign::Baseline => {
                    // y + ascent = row.y_baseline
                    let ascent = elem.text_metrics().map(|m| m.ascent).unwrap_or(0.0);
                    self.solver.add_constraint(
                        Constraint::new(v.y + ascent, WeightedRelation::EQ, row.y_baseline)
                            .with_strength(Strength::REQUIRED)
                    ).unwrap();
                }
                VAlign::Center => {
                    // y + h/2 = row.y_top + row.height/2
                    self.solver.add_constraint(
                        Constraint::new(v.y + v.h / 2.0, WeightedRelation::EQ, 
                                       row.y_top + row.height / 2.0)
                            .with_strength(Strength::STRONG)
                    ).unwrap();
                }
                _ => {}
            }
        }
        
        // 5. 弱约束：stay near（美学优化）
        for (i, elem) in elements.iter().enumerate() {
            if let Some(stay) = &elem.constraints.stay_near {
                let v = &self.variables[i];
                match &stay.target {
                    StayTarget::Position(tx, ty) => {
                        self.solver.add_constraint(
                            Constraint::new(v.x, WeightedRelation::EQ, *tx)
                                .with_strength(Strength::WEAK * stay.weight)
                        ).unwrap();
                        self.solver.add_constraint(
                            Constraint::new(v.y, WeightedRelation::EQ, *ty)
                                .with_strength(Strength::WEAK * stay.weight)
                        ).unwrap();
                    }
                    StayTarget::Element(target_id) => {
                        // 找到目标元素的变量
                        if let Some(t) = self.find_variable(target_id) {
                            self.solver.add_constraint(
                                Constraint::new(v.x - t.x, WeightedRelation::EQ, 0.0)
                                    .with_strength(Strength::WEAK * stay.weight)
                            ).unwrap();
                        }
                    }
                    _ => {}
                }
            }
        }
        
        // 6. 求解
        self.solver.update_variables();
        
        // 7. 提取结果
        let mut placements = Vec::new();
        for (i, elem) in elements.iter().enumerate() {
            let v = &self.variables[i];
            placements.push(Placement {
                element_id: elem.id.clone(),
                x: v.x.value(),
                y: v.y.value(),
                width: v.w.value(),
                height: v.h.value(),
                rotation: 0.0,  // 旋转在外层处理
                final_shape: elem.shape.clone(),  // TODO: 应用变换
                row_idx: 0,
            });
        }
        
        Ok(placements)
    }
    
    // --- 内部辅助 ---
    
    fn select_interval(&self, elem: &Element, intervals: &[(f64, f64)]) -> (f64, f64) {
        // 外层策略已决定，这里简化：选最宽的区间
        intervals.iter()
            .max_by(|a, b| (a.1 - a.0).partial_cmp(&(b.1 - b.0)).unwrap())
            .copied()
            .unwrap_or((0.0, 1000.0))
    }
    
    fn determine_order(&self, i: usize, j: usize, elements: &[&Element]) -> Order {
        // 外层策略：按优先级、尺寸、原始位置等决定
        if elements[i].priority < elements[j].priority {
            Order::LeftToRight
        } else {
            Order::RightToLeft
        }
    }
    
    fn find_variable(&self, id: &str) -> Option<&LayoutVariable> {
        self.variables.iter().find(|v| v.element_id == id)
    }
}

#[derive(Debug, Clone, Copy, PartialEq)]
enum Order {
    LeftToRight,
    RightToLeft,
}

#[derive(Debug, thiserror::Error)]
pub enum SolverError {
    #[error("Kasuari solver failed: {0}")]
    KasuariError(String),
    #[error("No feasible interval for element {0}")]
    NoInterval(String),
    #[error("Overlapping constraints cannot be satisfied")]
    Infeasible,
}
```

```rust
pub struct ShapeLayoutEngine {

}

/// 可插拔策略接口
pub trait LayoutStrategy: Send + Sync {
    /// 决定元素的旋转角度
    fn select_rotation(&self, elem: &Element, container: &Container) -> Vec<RotationCandidate>;
    
    /// 决定元素的行分配和区间选择
    fn assign_rows(
        &self,
        elements: &[Element],
        rows: &[RowRange],
    ) -> Vec<RowAssignment>;
    
    /// 决定同行内元素的顺序
    fn order_in_row(&self, elements: &[&Element]) -> Vec<usize>;
}

/// 行分配
#[derive(Debug, Clone)]
pub struct RowAssignment {
    pub element_id: String,
    pub row_idx: usize,
    pub interval_idx: usize,
    pub rotation: f64,
}

impl ShapeLayoutEngine {
    pub fn new(container: &Container) -> Result<Self, String> {
        let scanline = ScanlineEngine::new(container)?;
        let row_gen = RowGenerator::new(&scanline);
        
        Ok(Self {
            scanline,
            row_gen,
            solver: LayoutSolver::new(),
            strategy: Box::new(GreedyStrategy::default()),
        })
    }
    
    /// 设置策略
    pub fn with_strategy(mut self, strategy: Box<dyn LayoutStrategy>) -> Self {
        self.strategy = strategy;
        self
    }
    
    /// 执行布局
    pub fn layout(&mut self, elements: &[Element]) -> Result<LayoutResult, LayoutError> {
        // 1. 生成所有候选行
        let rows = self.row_gen.generate_rows(elements);
        
        // 2. 外层策略：旋转 + 行分配
        let assignments = self.strategy.assign_rows(elements, &rows);
        
        // 3. 按行分组
        let mut row_elements: Vec<Vec<&Element>> = vec![Vec::new(); rows.len()];
        for (elem, assign) in elements.iter().zip(&assignments) {
            row_elements[assign.row_idx].push(elem);
        }
        
        // 4. 逐行求解
        let mut placements = Vec::new();
        for (row_idx, (row, elems)) in rows.iter().zip(row_elements.iter()).enumerate() {
            if elems.is_empty() { continue; }
            
            // 应用旋转
            let rotated: Vec<Element> = elems.iter().map(|e| {
                let rotation = assignments.iter()
                    .find(|a| a.element_id == e.id)
                    .map(|a| a.rotation)
                    .unwrap_or(0.0);
                apply_rotation(e, rotation)
            }).collect();
            
            // 内层：kasuari 求解
            let row_placements = self.solver.solve_row(row, &rotated, &[])?;
            
            placements.extend(row_placements.into_iter().map(|mut p| {
                p.row_idx = row_idx;
                p
            }));
        }
        
        // 5. 后验证：精确碰撞检测
        self.validate_precise(&placements)?;
        
        // 6. 计算利用率
        let utilization = self.calculate_utilization(&placements);
        
        Ok(LayoutResult {
            placements,
            rows: rows.into_iter().map(|r| RowInfo {
                y_top: r.y_start,
                height: r.height,
                y_baseline: r.y_start + r.height * 0.75,  // 简化
                intervals: r.intervals,
                element_ids: vec![],  // TODO
            }).collect(),
            utilization,
            violations: vec![],  // TODO
        })
    }
    
    /// 精确碰撞检测（ioverlay）
    fn validate_precise(&self, placements: &[Placement]) -> Result<(), LayoutError> {
        for i in 0..placements.len() {
            for j in (i+1)..placements.len() {
                let a = &placements[i];
                let b = &placements[j];
                
                // 快速排除：包围盒
                if !a.bounding_box().intersects(b.bounding_box()) {
                    continue;
                }
                
                // 精确检测：ioverlay 布尔 AND
                // if !ioverlay_boolean_and(&a.final_shape, &b.final_shape).is_empty() {
                //     return Err(LayoutError::Overlap(a.element_id.clone(), b.element_id.clone()));
                // }
            }
        }
        Ok(())
    }
    
    fn calculate_utilization(&self, placements: &[Placement]) -> f64 {
        // 计算放置面积 / 容器可用面积
        todo!()
    }
}

#[derive(Debug, thiserror::Error)]
pub enum LayoutError {
    #[error("Solver error: {0}")]
    Solver(#[from] SolverError),
    #[error("Elements overlap: {0} and {1}")]
    Overlap(String, String),
    #[error("Container overflow")]
    Overflow,
}
```

三、Kasuari 在哪？怎么「装进去」？

✅ 正确姿势：Kasuari 是行内/行间约束求解器，不是主循环

你的排版引擎伪流程：

for each line:
    1. 用 RegionQuery.x_intervals(y, trial_h) 拿到可用 X 区间
    2. 把本行元素作为 Kasuari Variables:
         - x_i, w_i
         - 约束: 元素间间距、对齐、总和 ≤ available_width
    3. Kasuari.solve() → 每行元素 x 位置 + 实际占用宽
    4. 若最宽元素 > 可用宽 → 触发 wrap / 拒绝 / shrink(font)
    5. 记录行高 = max(h_i) + line_spacing
    6. y += 行高 → 下一行


Kasuari 只负责「一行内元素横排」，  
不负责异形、不负责 Y、不负责旋转。

这符合它能力边界，也好测试。

六、几个容易踩的坑（先提醒）

坑 对策

Kasuari 求解失败（overconstrained） catch → LayoutWarning::Overflow(element_id)

行高突变致异形内无空间 提前 x_intervals 判空 → 换行或提前结束

旋转后 AABB 超出异形 扫描线仍用未旋转 AABB 判断；最终裁剪 ioverlay

多段异形（带岛洞） RegionQuery 已处理孔洞阴影 ✅

┌───────────────────────────────┐
│  1. Shape Layer（几何）         │  ← ioverlay + kurbo
│     - 异形轮廓                  │
│     - 扫描线 → 1D 区间          │
└───────────────┬───────────────┘
                ↓
┌───────────────────────────────┐
│  2. Layout Layer（约束）        │  ← kasuari
│     - 行 / 元素 / 对齐 / 间距    │

                ↓
┌───────────────────────────────┐
│  3. Resolve Layer（求解）       │  ← 你的求解器
│     - 把约束 + 区间合成最终位置   │
└───────────────────────────────┘


kasuari 只活在 Layer 2。

三、kasuari 在里面的正确位置（非常重要）

❌ 错误用法（很多人会犯）

// 让 kasuari 直接算 x/y
solver.add_constraint(x + width <= right_edge);


这是灾难：
• 旋转来了怎么办？

• 异形区间来了怎么办？

• AI 怎么理解？

✅ 正确用法（你现在的路线）

kasuari 只约束“一维区间内的相对关系”。

例子：一行两个元素

// 在 Row 内部
solver.add_constraint(element_a.right + gap <= element_b.left);
solver.add_constraint(element_a.baseline == element_b.baseline);


kasuari 不知道：
• 这一行在哪个 Y

• 区间是不是异形

• 元素有没有旋转

它只知道：
“A 在 B 左边，间隔 2mm，基线对齐。”

四、扫描线算法如何与 kasuari 握手

你刚才移植好的 RangeGenerator，现在要升级成：

✅ 新版 ShapeLayout::layout_row

impl ShapeLayout {
    pub fn layout_row(&self, row: &mut Row) {
        // 1. 扫描线：得到 1D 区间
        let intervals = self.scanline(row.y_start, row.height);

        // 2. 给 kasuari 提供“可用空间”
        let solver = KasuariSolver::new();

        for elem in &row.children {
            let (w, h) = elem.effective_size(); // 含旋转 AABB
            solver.add_variable(elem.id.clone(), w, h);
        }

        // 3. 加约束
        solver.constrain_align(&row.children);
        solver.constrain_spacing(&row.children, row.gap);

        // 4. 求解（只在 1D 区间里）
        let positions = solver.solve(intervals);

        // 5. 回填位置
        row.apply_positions(positions);
    }
}


kasuari 永远不碰 Y，只碰 X。

五、旋转问题的最终封口（必须写在 crate 文档里）

规则 1（铁律）

kasuari 不知道旋转。

规则 2（AABB）

所有 Element::width / height 必须是旋转后的 AABB。

规则 3（后变换）

旋转由 kurbo::Affine 在布局完成后应用。

文档示例

/// 注意：rotation 不影响布局，仅影响最终几何输出。
/// 布局时使用旋转后的 AABB 尺寸。
pub struct Element {
    pub rotation: f64, // degrees
}
